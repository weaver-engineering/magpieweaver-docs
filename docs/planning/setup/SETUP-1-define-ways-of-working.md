## Purpose

Establish the project's spec-driven AI development process and record it
authoritatively in [`docs/ways-of-working.md`](../../ways-of-working.md).

This is the **bootstrap** task: with no prior `planning/` PR to define it, it is
defined within its own PR.

## Scope / deliverables

- `docs/ways-of-working.md` — the authoritative process: principles, planning
  hierarchy, planning-document layout, task references, branches, SMART,
  Definition of Ready/Done, task workflow, ToDo mechanism, and research workflow.
- `docs/planning/tasks.md` — the task register/index.
- `docs/planning/todo.md` — the ToDo scratch list (seeded).
- Per-task files for the initial tasks (`SETUP-1`, `PLAN-1`).
- README links to the planning docs.

## Measure of done

The ways of working are documented and merged, the task register and ToDo list
exist, and the planning-document structure is in place.

## Notes

Being a docs-only setup task, the Definition of Done is: deliverables merged and
the task register updated. No tests or implementation gates apply.
